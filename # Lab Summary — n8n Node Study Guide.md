# Lab Summary — n8n Node Study Guide

The most useful nodes in this lab were **AI Agent** and **When Chat Message Received** — together they form the core of any conversational workflow: the trigger exposes user input as structured JSON, while the AI Agent orchestrates the full reasoning loop by delegating to sub-nodes (OpenAI Chat Model, Simple Memory, SerpAPI). When picking nodes, the guiding question is: *what needs to happen to the data at this step?* — triggers start flows, data nodes load and reshape content, AI nodes reason or embed, vector nodes store and retrieve, tool nodes connect the agent to external services. The top debugging tip from this lab: **inspect the execution log node by node**, not the workflow as a whole — the Default Data Loader's pdf-parse v2 incompatibility only became visible at the individual node level, and would have been impossible to diagnose from the workflow-level error alone.
